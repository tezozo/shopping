## Commands

**Backend** (run from project root):
```bash
./gradlew bootRun                                              # start Spring Boot on :8080
./gradlew bootRun --args="--runDataInitializer"               # start and wipe+reseed all reference data
./gradlew test                                                 # unit tests only (Selenium excluded)
./gradlew test --tests "com.highspring.shopping.SomeTest"      # single test class
./gradlew seleniumTest                                         # Selenium e2e tests (builds frontend first)
./gradlew build                                                # compile + all tests
```

**Frontend** (run from `frontend/`):
```bash
npm install    # first time only
npm run dev    # Vite dev server on :5173 (proxies /api → :8080)
npm run build  # production build (output → frontend/dist/)
```

H2 console: http://localhost:8080/h2-console — JDBC URL `jdbc:h2:file:./data/shoppingdb`, user `sa`, no password.

## Data persistence

- Schema uses `ddl-auto=update` — the H2 file (`data/shoppingdb`) **survives restarts**.
- `DataInitializer` is a no-op unless `--runDataInitializer` is passed. When passed it clears all data and reseeds: 4 categories, 3 taxes, 3 discounts, 9 items, several category-discount links.
- Selenium tests use `@ActiveProfiles("test")` which switches to an in-memory H2 with `create-drop`; they seed their own minimal data.

## Architecture

### Backend — Spring Boot 3.3 / Java 21 / H2

`src/main/java/com/highspring/shopping/`:

| Package | Purpose |
|---|---|
| `entity/` | JPA entities |
| `repository/` | Spring Data JPA interfaces |
| `service/` | Business logic — only `ShoppingCartService` has meaningful logic |
| `controller/` | REST controllers, all prefixed `/api/`; `SpaController` forwards SPA routes to `index.html` |
| `dto/` | Request records: `CreateShoppingCartRequest`, `UpdateShoppingCartRequest`, `ShoppingCartItemRequest` |
| `config/` | `CorsConfig` (allows `:5173`), `DataInitializer` |
| `exception/` | `CartValidationException` + `GlobalExceptionHandler` (returns 400 with `{"error":"..."}`) |

**Key entity relationships:**
- `ShoppingCart` → `ShoppingCartItem[]` (OneToMany, cascade ALL, orphanRemoval, EAGER)
- `ShoppingCartItem` → `Item` (ManyToOne, EAGER)
- `Item` → `Set<Tax>`, `Set<Category>` (ManyToMany, EAGER, join tables `item_taxes` / `item_categories`)
- `CategoryDiscount` links `Category` ↔ `Discount` (a Discount's `end` field is mapped as `end_date` to avoid SQL keyword conflict)
- `ShoppingCart.id` is a UUID generated by Hibernate; all other PKs are `IDENTITY` longs

**Soft delete:** `ShoppingCart.deleted` flag + `deletedOn`; `findByOwnerAndDeletedFalse` and a `.filter(c -> !c.isDeleted())` guard in `getById` exclude them.

**Save validation:** `ShoppingCartService.update` throws `CartValidationException` if the submitted items span fewer than 3 distinct categories.

### Frontend — React 18 / Vite

Located in `frontend/`. Vite proxies `/api/*` to `:8080` (no CORS issues in dev).

| File | Purpose |
|---|---|
| `src/App.jsx` | Root — owns `owner` state (persisted in `localStorage`), renders routes |
| `src/components/OwnerInput.jsx` | Username gate; custom combobox that fetches existing owners from `GET /api/shopping-carts/owners` |
| `src/components/CartList.jsx` | Lists/creates/deletes carts for the current owner |
| `src/components/CartDetail.jsx` | Cart editor — computes discounts, taxes, net totals; shows inline save error |
| `src/api/carts.js` | Axios wrappers for all cart endpoints including `getOwners()` |
| `src/api/categoryDiscounts.js` | Fetches `GET /api/category-discounts` |

**Discount logic (in `CartDetail.jsx`):** active discounts are split into cumulative and non-cumulative. Best of (sum of all cumulative) vs (highest single non-cumulative) is applied per item. Taxes are calculated on the net (post-discount) amount.

### Tests

| Task | What runs | Needs |
|---|---|---|
| `./gradlew test` | Unit tests (`ShoppingCartServiceTest`) — Mockito only, no DB | nothing |
| `./gradlew seleniumTest` | `ShoppingCartSeleniumTest` — headless Chrome, full Spring context | Chrome installed |

Selenium tests use Page Objects in `src/test/java/com/highspring/shopping/selenium/page/`. The `seleniumTest` Gradle task builds the Vite frontend and adds it to the Spring Boot classpath so the full app is served from the random test port without a Vite dev server.
